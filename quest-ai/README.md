# Quest AI Character

Игровой ИИ-персонаж для квеста: языковая модель (DeepSeek или Mistral),
которая знает лор игры, отвечает в роли персонажа и подсказывает участникам
по ходу прохождения, не спойлеря будущие этапы.

## Как это работает (логика)

1. **Игрок пишет в чат** (`static/chat.html`) → сообщение уходит по WebSocket
   на сервер (`app/main.py`).

2. **Поиск релевантного лора (RAG).** Вопрос игрока превращается в embedding
   и ищется в векторной базе ChromaDB (`app/rag.py`). Важно: поиск **жёстко
   отфильтрован** по текущему этапу квеста — модель физически не получает
   в контекст материалы будущих этапов. Это надёжнее, чем просто написать
   в промпте "не спойлери": фильтрация происходит на уровне данных, а не
   поведения модели.

3. **Сборка промпта** (`app/prompts.py`). Собирается системный промпт:
   роль персонажа, стиль ответов, правила подсказок (намёк → прямой ответ
   только если команда застряла), и защита роли (игнорировать попытки
   выйти из персонажа/раскрыть системные инструкции).

4. **Запрос к LLM** (`app/llm_adapter.py`). Единый интерфейс `LLMAdapter`
   с двумя реализациями — `DeepSeekAdapter` и `MistralAdapter`. Переключение
   провайдера — одна переменная `LLM_PROVIDER` в `.env`, остальной код
   не меняется. Это позволяет протестировать оба варианта на одних и тех же
   сценариях и выбрать по факту (цена / качество роли / русский язык).

5. **Состояние игры** (`app/game_state.py`, SQLite). Хранится:
   - текущий этап (`stage`) — меняется вручную из админ-панели;
   - `session_id` — идентификатор текущего забега (одна команда за раз);
   - полная история сообщений — для контекста диалога и для лога ведущего.

6. **Админ-панель** (`/admin`): переключение этапа кнопками, сброс сессии
   перед новой командой (без потери логов предыдущей), просмотр диалога
   в реальном времени. Защищена паролем из `ADMIN_PASSWORD` в `.env`
   (браузер спросит пароль при первом заходе и передаёт его заголовком
   `X-Admin-Password` на все запросы `/admin/*`).

## Подготовка лора

Разложите материалы по папкам `lore/stage_N/` — номер папки определяет,
на каком этапе квеста эта информация становится доступна персонажу:

```
lore/
    stage_1/
        introduction.docx
        location_hall.pdf
    stage_2/
        location_library.docx
        hint_locked_box.pdf
    stage_3/
        ...
```

Поддерживаются `.docx` и `.pdf`. После добавления/изменения файлов запустите
индексацию заново:

```bash
python ingest.py
```

Скрипт очищает старую версию базы и переиндексирует всё с нуля — так вы
не получите дублей при повторных запусках.

## Запуск

### Локально (для разработки/тестов)

```bash
python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# впишите в .env свой DEEPSEEK_API_KEY или MISTRAL_API_KEY

python ingest.py               # индексация лора
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Чат для игроков: `http://localhost:8000/`
Админ-панель: `http://localhost:8000/admin`

### Через Docker (рекомендуется для игрового дня)

```bash
cp .env.example .env
# впишите ключи API

docker compose up --build -d
python ingest.py    # если запускаете вне контейнера — либо
docker compose exec quest-ai python ingest.py   # индексация внутри контейнера
```

## Переключение между командами (один забег за раз)

Перед стартом новой команды нажмите **"Сбросить сессию"** в админ-панели:
- этап возвращается на 1;
- создаётся новый `session_id`, история для игроков начинается с чистого листа;
- лог предыдущей команды остаётся в базе (можно посмотреть по старому
  `session_id` напрямую в `data/game.db`, если понадобится для разбора игры).

## Смена провайдера LLM

В `.env`:
```
LLM_PROVIDER=deepseek   # или mistral
```
Оба провайдера работают через один и тот же интерфейс — код переписывать
не нужно, только ключ и модель в `.env`.

## Сетевые требования

Игровому компьютеру нужен исходящий доступ к:
- `https://api.deepseek.com` (если используете DeepSeek)
- `https://api.mistral.ai` (если используете Mistral)

Если хотите ограничить компьютер только этими адресами (белый список) —
это настраивается на уровне роутера/файрвола игровой сети, приложение
на это никак не завязано.

## Возможные доработки

- Автоматическое определение прогресса по ключевым словам в диалоге
  (как подсказка ведущему, не как единственный триггер смены этапа).
- Модерация/фильтр на выходе модели, если нужен дополнительный уровень
  защиты от нежелательных ответов.
- Логирование в текстовый файл/экспорт для разбора игры после сессии.
